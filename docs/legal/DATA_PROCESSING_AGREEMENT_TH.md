## 1. ขอบเขตของการประมวลผล

MarSci Diagnostics อาจประมวลผลข้อมูลส่วนบุคคลเท่าที่จำเป็นอย่างสมเหตุสมผลเพื่อให้บริการตามที่ลูกค้าร้องขอ รวมถึงการบริหารบัญชี การยืนยันตัวตน การดำเนินงานด้านการเรียกเก็บเงิน การสนับสนุนการเชื่อมต่อแพลตฟอร์ม การประมวลผลเชิงวินิจฉัย และกิจกรรมสนับสนุนที่เกี่ยวข้อง

การประมวลผลจำกัดอยู่ภายใต้ความสัมพันธ์ในการให้บริการระหว่างลูกค้ากับ MarSci Diagnostics และดำเนินการตามคำสั่งที่มีการบันทึกของลูกค้า เว้นแต่การประมวลผลนั้นจำเป็นตามกฎหมายที่ใช้บังคับ

## 2. ประเภทของข้อมูลส่วนบุคคลที่ประมวลผล

ขึ้นอยู่กับวิธีใช้งานของลูกค้า ข้อมูลส่วนบุคคลที่อาจถูกประมวลผล ได้แก่:

- ชื่อ
- อีเมล
- รหัสบัญชี
- ข้อมูลยืนยันตัวตนจาก Google OAuth
- ข้อมูลอ้างอิงลูกค้า Stripe และ metadata ด้านการเรียกเก็บเงินที่เกี่ยวข้อง
- metadata ของการเชื่อมต่อแพลตฟอร์มหรือบัญชี
- log การใช้งานและบันทึกการติดต่อ support
- ข้อมูลประสิทธิภาพการตลาดที่อาจมีตัวระบุหรือ metadata ที่เชื่อมโยงกับบัญชี

MarSci Diagnostics ไม่มีเจตนาประมวลผลรายละเอียดบัตรชำระเงินแบบเต็มโดยตรง Stripe เป็นผู้จัดการการประมวลผลข้อมูลบัตรชำระเงิน

## 3. ประเภทของเจ้าของข้อมูล

ประเภทของเจ้าของข้อมูลอาจรวมถึง:

- ผู้ใช้และผู้ดูแลระบบของลูกค้า
- บุคลากรที่ได้รับอนุญาตของลูกค้า
- ผู้ปฏิบัติงานบัญชีแพลตฟอร์ม
- ผู้ติดต่อด้าน support
- บุคคลซึ่งตัวระบุอาจปรากฏในชุดข้อมูลการตลาดที่เชื่อมโยงกับบัญชี แล้วแต่กรณี

## 4. หน้าที่ของผู้ประมวลผลข้อมูล

MarSci Diagnostics จะ:

- ประมวลผลข้อมูลส่วนบุคคลเพื่อวัตถุประสงค์ในการให้บริการที่ตกลงกันไว้เท่านั้น
- ไม่นำข้อมูลของลูกค้าไปใช้เพื่อวัตถุประสงค์อื่นที่ไม่เกี่ยวข้อง เว้นแต่กฎหมายหรือสัญญาอนุญาต
- รักษาข้อผูกพันด้านความลับที่เหมาะสมสำหรับบุคลากรและผู้รับจ้างที่เข้าถึงข้อมูลดังกล่าว
- ช่วยเหลือลูกค้าเท่าที่สมเหตุสมผลเกี่ยวกับคำขอใช้สิทธิของเจ้าของข้อมูล
- ช่วยเหลือเกี่ยวกับคำขอข้อมูลเพื่อการปฏิบัติตามกฎหมายในขอบเขตที่มีข้อมูลและทำได้อย่างสมเหตุสมผล
- แจ้งลูกค้าหากคำสั่งดูมีแนวโน้มขัดต่อกฎหมายที่ใช้บังคับ เมื่อเหมาะสมและสามารถทำได้อย่างสมเหตุสมผล

## 5. มาตรการคุ้มครองความปลอดภัย

MarSci Diagnostics ใช้มาตรการทางการบริหาร ทางเทคนิค และทางองค์กรที่มีวัตถุประสงค์เพื่อช่วยคุ้มครองข้อมูลส่วนบุคคล มาตรการดังกล่าวอาจรวมถึงการควบคุมการเข้าถึง การยืนยันตัวตน การจัดเก็บ log การจำกัดสิทธิ์ตามบทบาท และแนวปฏิบัติด้านความมั่นคงปลอดภัยที่เหมาะสมกับลักษณะของบริการ

MarSci Diagnostics ไม่รับประกันความปลอดภัยอย่างสมบูรณ์ และมาตรการอาจมีการเปลี่ยนแปลงตามกาลเวลา

## 6. ผู้ประมวลผลข้อมูลช่วง

MarSci Diagnostics อาจใช้ผู้ประมวลผลข้อมูลช่วงและผู้ให้บริการเพื่อดำเนินงานของบริการ รวมถึงผู้ให้บริการด้าน:

- การโฮสต์และ cloud infrastructure
- การยืนยันตัวตน
- การเรียกเก็บเงิน
- การจัดเก็บ log
- การวิเคราะห์
- การส่งอีเมล
- เครื่องมือ support

ผู้ประมวลผลข้อมูลช่วงในปัจจุบันหรืออนาคตอาจรวมถึงผู้ให้บริการอย่าง Google, Stripe และผู้ให้บริการ cloud infrastructure แล้วแต่กรณี รายละเอียดเพิ่มเติมอาจจัดให้ต่างหากเมื่อมีความพร้อม

## 7. การโอนข้อมูลระหว่างประเทศ

ข้อมูลของลูกค้าอาจถูกประมวลผลหรือจัดเก็บไว้นอกประเทศไทย หรือนอกเขตอำนาจของลูกค้าเอง ทั้งนี้ขึ้นอยู่กับตำแหน่งของโครงสร้างพื้นฐานและผู้ให้บริการ MarSci Diagnostics อาจอาศัยผู้ให้บริการที่มีการประมวลผลข้อมูลข้ามพรมแดน

ส่วนนี้อาจต้องได้รับการทบทวนทางกฎหมายเพิ่มเติมตามตำแหน่งของลูกค้าและการโฮสต์จริง

## 8. การแจ้งเหตุข้อมูลรั่วไหล

หาก MarSci Diagnostics ทราบถึงเหตุข้อมูลส่วนบุคคลรั่วไหลที่ได้รับการยืนยันและมีผลกระทบต่อข้อมูลของลูกค้า MarSci Diagnostics อาจแจ้งลูกค้าโดยไม่ชักช้าเกินสมควร โดยพิจารณาจากข้อมูลที่มีอยู่ในขณะนั้นและภาระผูกพันตามกฎหมายที่เกี่ยวข้อง

การแจ้งครั้งแรกอาจมีข้อมูลจำกัด และอาจมีการแจ้งเพิ่มเติมภายหลังเมื่อมีข้อมูลมากขึ้น

## 9. การลบหรือคืนข้อมูลเมื่อสิ้นสุดสัญญา

เมื่อบริการที่เกี่ยวข้องสิ้นสุดลง MarSci Diagnostics อาจลบหรือคืนข้อมูลของลูกค้าเมื่อมีข้อกำหนดตามสัญญา มีความพร้อมทางเทคนิค และไม่ขัดต่อกฎหมาย อย่างไรก็ตาม ข้อมูลบางส่วนอาจยังต้องเก็บไว้เพื่อเหตุผลด้านกฎหมาย บัญชี การตรวจสอบ ความมั่นคงปลอดภัย การป้องกันการทุจริต หรือความสมบูรณ์ของการดำเนินงาน

กระบวนการลบหรือคืนข้อมูลขึ้นอยู่กับการพัฒนาทางเทคนิคจริงและข้อจำกัดตามกฎหมายที่มีผลในขณะนั้น
