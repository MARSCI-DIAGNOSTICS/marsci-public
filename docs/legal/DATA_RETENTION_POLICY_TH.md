# นโยบายการเก็บรักษาข้อมูล

เอกสารนี้เป็นคำแปลภาษาไทยของฉบับภาษาอังกฤษ
หากมีความไม่สอดคล้องกัน ให้ยึดฉบับภาษาอังกฤษเป็นหลัก

อัปเดตล่าสุด: 6 มีนาคม 2026

นโยบายการเก็บรักษาข้อมูลฉบับนี้เป็นร่างสำหรับ MarSci Diagnostics โดยอธิบายแนวทางที่ตั้งใจใช้ในปัจจุบันสำหรับหมวดข้อมูลสำคัญของบริการแบบ SaaS ระยะเวลาการเก็บรักษาขั้นสุดท้ายอาจเปลี่ยนแปลงได้ตามข้อกำหนดทางกฎหมาย การเปลี่ยนแปลงของโครงสร้างพื้นฐาน และรายละเอียดการพัฒนาผลิตภัณฑ์

## 1. หลักการเก็บรักษาข้อมูล

MarSci Diagnostics อาจเก็บข้อมูลไว้เท่าที่จำเป็นอย่างสมเหตุสมผลเพื่อ:

- ให้บริการ
- รักษาความต่อเนื่องของบัญชี
- สนับสนุนการเรียกเก็บเงินและงานบัญชี
- รักษาความมั่นคงปลอดภัยและความสมบูรณ์ของการดำเนินงาน
- แก้ไขปัญหา
- ปฏิบัติตามกฎหมาย
- บังคับใช้สิทธิตามสัญญาเมื่อเกี่ยวข้อง

## 2. ข้อมูลบัญชีผู้ใช้

ตัวอย่าง:

- ชื่อ
- อีเมล
- รหัสผู้ใช้
- ข้อมูลโปรไฟล์พื้นฐาน

แนวทางการเก็บรักษา:

- เก็บไว้ตราบเท่าที่บัญชียังใช้งานอยู่; และ
- อาจเก็บต่ออีกระยะเวลาที่สมเหตุสมผลเพื่อการกู้คืนบัญชี support การตรวจสอบ ข้อพิพาท หรือการปฏิบัติตามกฎหมาย

## 3. ข้อมูลเมตาการยืนยันตัวตนแบบ OAuth

ตัวอย่าง:

- รหัสบัญชี Google OAuth
- ข้อมูลการเชื่อมโยงบัญชี
- เวลาในการยืนยันตัวตนหรือ metadata ที่เกี่ยวข้อง

แนวทางการเก็บรักษา:

- เก็บไว้ตราบเท่าที่จำเป็นต่อการเชื่อมโยงบัญชี การเข้าถึง และความมั่นคงปลอดภัย; และ
- อาจเก็บต่อเมื่อจำเป็นเพื่อเหตุผลด้านความมั่นคงปลอดภัย การตรวจสอบ หรือการป้องกันการทุจริต

## 4. ข้อมูลการเรียกเก็บเงิน

ตัวอย่าง:

- ข้อมูลอ้างอิงลูกค้า Stripe
- ข้อมูลอ้างอิง checkout session
- รหัส subscription
- metadata ของสถานะ invoice และการชำระเงิน

แนวทางการเก็บรักษา:

- เก็บไว้ตามความจำเป็นสำหรับการเรียกเก็บเงิน งานบัญชี ภาษี การตรวจสอบ และการเก็บหลักฐานตามกฎหมาย; และ
- อาจเก็บต่อหลังปิดบัญชีเมื่อกฎหมายกำหนดหรือมีความจำเป็นทางปฏิบัติการ

## 5. ข้อมูลเมตาของการเชื่อมต่อแพลตฟอร์ม

ตัวอย่าง:

- รหัสระดับบัญชี
- metadata ของการเชื่อมต่อ
- เวลาในการเชื่อมต่อ

แนวทางการเก็บรักษา:

- เก็บไว้ตราบเท่าที่จำเป็นต่อการทำงานของบริการ การเชื่อมโยงบัญชี support และการแก้ไขปัญหา; และ
- อาจถูกลบหรือทำให้เหลือน้อยที่สุดเมื่อไม่จำเป็นอีกต่อไป ภายใต้ข้อจำกัดทางเทคนิคและกฎหมาย

## 6. ชุดข้อมูล KPI ทางการตลาดและข้อมูลการวินิจฉัย

ตัวอย่าง:

- ชุดข้อมูล KPI ที่อัปโหลดหรือเชื่อมต่อ
- ประวัติ input เพื่อการวินิจฉัย
- ประวัติผลลัพธ์การวินิจฉัย

แนวทางการเก็บรักษา:

- เก็บไว้ตราบเท่าที่จำเป็นเพื่อให้บริการ รักษาบริบทประวัติการวินิจฉัย สนับสนุนการแก้ไขปัญหา และดำเนินงานของผลิตภัณฑ์; และ
- อาจเปลี่ยนแปลงได้ตามรูปแบบการจัดเก็บ การควบคุมของลูกค้า และเครื่องมือลบข้อมูลในอนาคต

## 7. Logs

ตัวอย่าง:

- access logs
- security logs
- operational logs
- error logs

แนวทางการเก็บรักษา:

- เก็บไว้ในช่วงเวลาจำกัดตามความเหมาะสมต่อความมั่นคงปลอดภัยของบริการ การป้องกันการใช้งานในทางที่ผิด การแก้ไขปัญหา และการติดตามการทำงานของระบบ; และ
- ระยะเวลาการเก็บอาจแตกต่างกันตามระบบและระดับความสำคัญของเหตุการณ์

## 8. ข้อยกเว้นและการเก็บรักษาตามกฎหมาย

ข้อมูลบางส่วนอาจถูกเก็บไว้นานขึ้นหากกฎหมายกำหนด หรือจำเป็นต่อข้อพิพาท การตรวจสอบ การป้องกันการทุจริต หรือการตอบสนองด้านความมั่นคงปลอดภัย

## 9. การเปลี่ยนแปลงนโยบาย

นโยบายนี้อาจมีการปรับปรุงเมื่อโครงสร้างพื้นฐาน พฤติกรรมของผลิตภัณฑ์ หรือข้อกำหนดทางกฎหมายเปลี่ยนแปลงไป
